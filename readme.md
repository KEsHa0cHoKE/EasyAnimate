# 🎨 Anim — Анимация переменных для GameMaker

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)
[![GMS 2.3+](https://img.shields.io/badge/GMS-2.3%2B-blue.svg)](https://gamemaker.io)

Простая, легковесная и мощная система анимации переменных для **GameMaker Studio 2.3+**. 
Позволяет плавно изменять любые числовые свойства объектов и структур, используя встроенные `time_source`, кривые анимации и режим ключевых кадров.

## ✨ Возможности
- 🎯 **Два режима:** анимация к одному значению или по массиву ключевых точек (Path)
- ⏱️ **Гибкое время:** `ANIM_UNITS.FRAMES` или `ANIM_UNITS.SECONDS`
- 📈 **Кривые:** полная поддержка `AnimCurve` ассетов + экстраполяция (bounce, overshoot)
- ⏸️ **Управление:** `pause`, `resume`, `stop`, `togglePause`
- 🔔 **Коллбеки:** на завершение анимации и на каждый промежуточный ключ
- 🔒 **Инкапсуляция:** приватные данные скрыты в структуре `private`
- 🧪 **Авто-тесты:** встроенный набор юнит-тестов для проверки стабильности
- 📦 **Нулевые зависимости:** работает на чистом GML, без сторонних библиотек

## 📦 Установка
1. Импортируйте скрипты `Anim` и `anim_is_active` в ваш проект.
2. (Опционально) Импортируйте `test_Anim` для запуска автотестов.
3. Готово! Система не требует дополнительных настроек или инициализации.

## 🚀 Быстрый старт

```gml
// 1. Анимация одной переменной к целевому значению
var anim = new Anim(self, nameof(x));
anim.start(500, 1.5, ANIM_CURVE_EASE_IN_OUT, ANIM_UNITS.SECONDS);

// 2. Анимация нескольких переменных одновременно
var anim2 = new Anim(self, [nameof(x), nameof(y)]);
anim2.start(200, 0.5, ANIM_CURVE_LINEAR, ANIM_UNITS.FRAMES);

// 3. Анимация по ключевым точкам (Path-режим)
var anim3 = new Anim(self, nameof(alpha));
anim3.start([1, 0.5, 1, 0.5, 0], 2, ANIM_CURVE_SINE, ANIM_UNITS.SECONDS);
```

## 📖 Документация по API

### 🏗️ Конструктор
```gml
Anim(target, vars)
```
| Параметр | Тип | Описание |
|----------|-----|----------|
| `target` | `instance` / `struct` | Объект или структура, чьи свойства будут анимироваться |
| `vars` | `string` / `array<string>` | Имя переменной или массив имён. Для инстансов используйте `nameof()`, для структур — строковые литералы `"field"` |

### ⚙️ Методы управления

| Метод | Описание |
|-------|----------|
| `start(endValue, duration, curve = ANIM_CURVE_LINEAR, units = ANIM_UNITS.SECONDS, perKeyframe = false)` | Запуск анимации. Если `endValue` — массив (`length > 1`), автоматически активируется Path-режим |
| `stop()` | Остановка анимации. Переменные остаются на текущем значении |
| `pause()` | Поставить анимацию на паузу |
| `resume()` | Снять с паузы |
| `togglePause()` | Переключить состояние паузы |
| `getProgress()` | Возвращает текущий прогресс (`0.0` – `1.0+`) |
| `isActive()` | `true`, если анимация выполняется |
| `isPaused()` | `true`, если анимация на паузе |

### 🔔 Коллбеки
| Метод | Описание |
|-------|----------|
| `onComplete(callback, deleteAfterCall = false)` | Установить функцию, которая вызовется в конце анимации |
| `onKeyframe(index, callback, deleteAfterCall = false)` | (Path-режим) Коллбек при достижении ключа с индексом `index`. Используйте `ANIM_END` для конца |
| `clearOnComplete()` | Очистить коллбек завершения |
| `clearKeyframeCallback(index)` | Очистить коллбек для конкретного ключа |

### 📐 Перечисления и макросы
```gml
enum ANIM_UNITS { FRAMES, SECONDS }
#macro ANIM_END -1
#macro __ANIM_USE_DELTA false // Измените на true для поддержки переменной частоты кадров
```

## 🛠️ Продвинутое использование

### 🔁 Path-режим (анимация по ключевым точкам)
Если передать массив в `start()`, система автоматически создаст внутренний `path` и будет плавно проходить по всем точкам:
```gml
anim.start([100, 300, 150, 500], 3, ANIM_CURVE_SINE, ANIM_UNITS.SECONDS);

// Коллбек на второй ключ (индекс 1)
anim.onKeyframe(1, function() { show_debug_message("Ключ 1 достигнут!"); });
```
Параметр `perKeyframe = true` заставляет `duration` применяться к *каждому сегменту* отдельно, а не ко всей анимации целиком.

### ⏱️ Дельта-тайм
По умолчанию анимация привязана к фиксированному кадру (`time_source_units_frames`). 
Для поддержки переменной частоты кадров измените макрос в начале скрипта:
```gml
#macro __ANIM_USE_DELTA true
```

### 🔍 Проверка активности переменной
```gml
if (anim_is_active(self, nameof(x))) {
    // x сейчас анимируется другим экземпляром Anim
}
```

## 🧪 Автотесты
В репозиторий включён набор юнит-тестов. Для запуска создайте временный объект и в `Create` добавьте:
```gml
if (!test_Anim_run_all()) {
    show_message("⚠️ Некоторые тесты не пройдены. См. консоль.");
}
```
Результаты выводятся в консоль с цветовой индикацией. Все тесты изолированы, не влияют на состояние проекта и проверяют ключевые сценарии работы конструктора.

## ⚠️ Важные замечания
- Требуется **GameMaker Studio 2.3.0+** (поддержка `constructor`, `private`, `time_source`)
- `nameof()` работает только для переменных в области видимости инстанса. Для структур передавайте строки: `new Anim(myStruct, "hp")`
- `call_later` работает асинхронно. Коллбеки `onComplete`/`onKeyframe` срабатывают на следующем обработанном кадре, а не мгновенно
- При уничтожении `target` анимация автоматически останавливается и очищает ресурсы
- Встроенный `path` создаётся динамически и удаляется после завершения анимации. Не требует ручного управления памятью
